# Third-party license review

검토일: 2026-07-20

이 저장소의 자체 소스는 [MIT License](LICENSE)로 배포됩니다. 설치되는 외부 패키지는 각각의 라이선스를 유지하며, 이 문서는 `pnpm-lock.yaml`의 모든 플랫폼 선택 항목까지 확장해 확인한 결과와 재배포 시 검토할 지점을 기록합니다. 법률 자문을 대신하지 않습니다.

## 검토 방법

```bash
pnpm install --frozen-lockfile
pnpm licenses list --json
```

위 명령으로 확인한 macOS arm64 설치 트리는 고유 패키지명 464개, 이름·버전 조합 495개였습니다. 잠금 파일의 다른 운영체제·아키텍처 선택 패키지까지 포함한 696개 항목의 SPDX 표현은 다음과 같습니다.

## 직접 의존성

런타임 직접 의존성은 MIT 또는 Apache-2.0입니다. 개발 도구의 직접 의존성도 MIT, Apache-2.0, 또는 `MIT OR Apache-2.0`로 확인했습니다. 따라서 외부 패키지의 코드를 이 저장소에 복사하지 않고 패키지 관리자로 설치하는 현재 소스 배포 방식은 프로젝트의 MIT 라이선스와 충돌하는 항목이 확인되지 않았습니다.

## 재배포 시 주의할 항목

- `@img/sharp-libvips-*`의 플랫폼별 선택 패키지는 `LGPL-3.0-or-later`로 표시됩니다. 이 저장소는 `node_modules`나 해당 네이티브 라이브러리를 포함하지 않습니다. 컨테이너, 데스크톱 번들 또는 설치 이미지에 바이너리를 함께 배포한다면 해당 LGPL 원문, 저작권 고지, 사용한 소스 또는 적법한 소스 제공 방식을 함께 검토해야 합니다.
- `caniuse-lite` 데이터는 `CC-BY-4.0`으로 표시됩니다. 해당 데이터를 별도 산출물로 재배포하거나 변형해 포함할 경우 패키지의 저작자 표시와 라이선스 고지를 유지해야 합니다.
- `MPL-2.0` 패키지를 수정해 그 파일 자체를 재배포한다면 수정한 MPL 파일의 소스 제공 의무를 별도로 확인해야 합니다. 현재 저장소에는 해당 패키지 소스를 복사하거나 수정해 포함하지 않습니다.
- TypeScript의 `ThirdPartyNoticeText.txt`, axe-core의 `LICENSE-3RD-PARTY.txt`처럼 패키지 안에 별도 고지가 있는 경우 해당 패키지를 빌드·컨테이너에 포함해 배포할 때 함께 유지해야 합니다.
- 잠금 파일은 설치 버전과 무결성 정보를 고정할 뿐, 외부 패키지의 라이선스를 프로젝트 라이선스로 변경하지 않습니다. 의존성을 갱신할 때 위 명령을 다시 실행하고 새 SPDX 표현이나 `UNKNOWN` 항목을 검토해야 합니다.

패키지별 원문과 저작권 고지는 설치된 각 패키지의 `LICENSE`, `COPYING`, `NOTICE`, `package.json` 및 연결된 공식 저장소를 기준으로 확인하세요.
