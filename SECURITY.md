# Security policy

## 지원 범위

보안 수정은 기본 브랜치의 최신 버전을 기준으로 진행합니다. 공개 릴리스 태그가 생기기 전까지 별도 장기 지원 버전은 운영하지 않습니다.

## 취약점 제보

민감한 취약점은 공개 이슈에 세부 내용을 남기지 말고 저장소의 비공개 보안 제보 기능을 사용해 주세요. 다음 정보만 최소 범위로 포함하면 재현에 도움이 됩니다.

- 영향을 받는 커밋 또는 버전
- 민감정보를 제거한 재현 절차
- 예상 동작과 실제 동작
- 가능한 경우 최소 샘플 프로젝트

실제 API 키, 사내 저장소, 개인정보, 내부 URL과 전체 로그는 첨부하지 마세요.

## 신뢰 경계

로컬 정적 분석은 브라우저 안에서 실행되지만 선택적 LLM 기능은 마스킹된 질문·대화·상대 경로·그래프·근거 excerpt를 서버와 설정된 외부 API로 전송합니다. 마스킹은 완전한 DLP가 아니므로 민감 저장소에는 조직의 접근 제어와 별도 데이터 유출 방지 정책을 적용해야 합니다.
